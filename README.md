# 🥘 魔法厨房 — 五色食物搭配师

> 面向小学生的中医食育互动游戏，纯前端 HTML，双击即用，投屏友好。

## 🎮 玩法

1. 从 40 种食材中挑选 **1~5 种**
2. 点击 **🔥 开始烹饪**
3. 获得创意菜名 + 评分 + 养生建议

## ✨ 功能

| 功能 | 说明 |
|------|------|
| 食材选择 | 40 种食材按五行五色分类，点击加入锅中 |
| 评分系统 | 口味 40% + 营养 20% + 创意 10%，满分 100 |
| 创意模式 | 可选开关，20 动词 × 17 风格词 × 10 前缀 → 千变菜名 |
| 经典名菜 | 命中 20 道预设配对直接出经典菜名（番茄豆腐汤等） |
| 五行分析 | 展示食材五行关系（不计入评分） |
| 烹饪记录 | 右侧栏历史列表，点击可回顾 |
| 收藏系统 | ❤️ 收藏心仪菜品，侧栏独立查看 |
| 成就徽章 | 6 枚成就（烹饪 10 次、满分、五色齐全等） |
| 投屏优化 | 1366×768 全屏可见，大字清晰 |

## 📊 评分算法

```
总分 = (经典×0.35 + 五色×0.25 + 创意×0.15 + 稀有×0.10 + 基础×0.15) × 13.3 ± 3
```

五维评分体系（每维 0-10 分）：

| 维度 | 权重 | 规则 |
|------|:---:|------|
| 经典匹配 | 35% | 经典名菜=10, 单品菜=8, 2种=7, 3种=6, 4~5种=5 |
| 五色均衡 | 25% | 1+颜色数×1.8（1色=2.8 → 5色=10） |
| 创意维度 | 15% | 颜色数×2（1色=2 → 5色=10） |
| 稀有食材 | 10% | 基础4 + 稀有+3 + 五色齐全+3，上限10 |
| 基础保底 | 15% | 恒为5，保证最低分 |

**设计理念**：经典2种口味好但颜色少 → 五色维度拖低；5色搭配口感降但颜色维度拉高 → **两者在80-100区间公平竞争**。

| 搭配 | ≈分数 |
|------|:---:|
| 5 色稀有搭配 | **97-100** |
| 经典 2 种（番茄+豆腐） | 82-88 |
| 4 种 4 色均衡 | 75-81 |
| 3 种 3 色 | 77-83 |
| 单品菜（清炒西蓝花） | 63-69 |
| 2 种随机搭配 | 68-74 |
| 5 种同色堆叠 | 49-55 |

## 🎨 命名引擎

### 标准模式
模板命名：`{烹饪法} + {主要食材} + {次要食材}`

### ✨ 创意模式
扩展词库：20 烹饪法 × 17 风格词 × 10 前缀 × 10 后缀 × 15 诗意词

```
标准：番茄炒豆腐
创意：限定番茄糖醋豆腐 / 私房番茄烩玛瑙豆腐 / 朝露番茄双品
```

## 💾 数据存储

全部使用 `localStorage`，浏览数据不离开本机：

| Key | 内容 |
|-----|------|
| `magic_kitchen_history` | 烹饪记录（最多 30 条） |
| `magic_kitchen_favorites` | 收藏菜品（最多 20 条） |
| `magic_kitchen_stats` | 统计数据（次数、最高分） |
| `magic_kitchen_achievements` | 成就徽章 |
| `magic_kitchen_creative` | 创意模式开关状态 |

## 🔧 技术栈

- 原生 HTML + CSS + JavaScript（ES5 兼容）
- CSS Grid + Flexbox 响应式布局
- localStorage 数据持久化

## 📝 设计决策

| 决策 | 理由 |
|------|------|
| 五行不计入评分 | 用户要求，避免小学生困惑 |
| 口味权重随食材数递减 | 防止盲目选满 5 种，单品也可高分 |
| 拒绝 ONNX 浏览器 AI | 英文模型不支持中文，中文模型过大（>1GB） |
| 创意模式替代 AI | 纯模板引擎，零延迟零网络，实际效果不输小模型 |
| 投屏优先大字布局 | 1366×768 无滚动，2 列大方块，最小字号 15px |
