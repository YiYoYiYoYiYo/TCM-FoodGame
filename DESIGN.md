# 魔法厨房 · 五色食物搭配游戏 — 功能设计草案

## 1. 产品定位

| 维度 | 说明 |
|------|------|
| **名称** | 魔法厨房 — 五色食物搭配师 |
| **slogan** | 2 种食材 × 无限可能 = 你的专属五行佳肴 |
| **场景** | 小学课堂互动 / 学生课后自主探索 |
| **年龄** | 8-12 岁小学生 |
| **形态** | 纯静态 HTML 网页，无需后端，双击即开 |
| **风格** | 童趣国潮（暖黄 + 草绿 + 朱红，圆角卡片，emoji 食材） |

---

## 2. 核心玩法流程

```
[ 进入厨房 ] → [ 从 40 种食材中选 2 种 ] → [ 点击 "开始烹饪" ]
     → [ 展示结果卡片：菜名 + 五行分析 + 好吃评分 + 健康小贴士 ]
     → [ 可收藏 / 重新搭配 ]
```

### 2.1 分步说明

| 步骤 | 学生操作 | 系统反馈 |
|------|----------|----------|
| **Step 1** | 浏览 5 色食材区，了解每种食材的属性 | 食材卡片显示：emoji、名称、五行色标签、归经 |
| **Step 2** | 点击第 1 种食材（高亮选中） | 对应颜色区闪烁，顶部"锅"中显示食材 1 |
| **Step 3** | 点击第 2 种食材（高亮选中） | 锅中有 2 种食材，"开始烹饪"按钮亮起 |
| **Step 4** | 点击 "🔥 开始烹饪" | 动画：食材飞入锅中，冒烟特效 |
| **Step 5** | 查看结果 | 弹出/展开结果卡片（菜名 + 五行分析 + 评分 + 贴士） |
| **Step 6** | 收藏或继续 | 可收藏到"我的菜谱"，可点击"重新搭配" |

## 4. 匹配规则引擎（菜品生成逻辑）

### 4.1 匹配类型

| 类型 | 判定条件 | 示例 | 出现概率 |
|------|---------|------|---------|
| **经典名菜** | 特定食材对硬编码匹配 | 鸡肉+蘑菇 → 小鸡炖蘑菇 | — |
| **五行相生** | 五行属性为母→子关系（木→火→土→金→水→木） | 绿色(木)+红色(火) → "木火通明"创意菜 | 约 30% 命中 |
| **五行相克** | 五行属性为克→被克关系（木克土、火克金、土克水、金克木、水克火） | 绿色(木)+黄色(土) → "土木大战"黑暗料理 | 约 30% 命中 |
| **同色系** | 两种食材同色 | 番茄+草莓 → "红色双响" | 约 20% |
| **随机创意** | 其余组合 | 生成创意菜名 | 约 20% |

### 4.3 评分体系

| 评分维度 | 权重 | 说明 |
|----------|------|------|
| **口味契合度** | 40% | 经典名菜=5分，五行相生=4分，同色=3分，五行相克=1-2分 |
| **五行和谐度** | 30% | 相生=5分，同属性=3分，相克=1分 |
| **营养均衡** | 20% | 不同色=+2，含蛋白=+1，含维C=+1 |
| **创意加分** | 10% | 名字有趣=+1，食材稀有度=+1 |

总分 = 加权平均 × 20 → 得到 1-100 的"好吃指数"

## 7. 交互细节设计

### 7.1 食材卡片交互

| 状态 | 视觉效果 |
|------|---------|
| **默认** | 圆角卡片，浅底色，emjoi+名称 |
| **悬停** | 放大 110%，阴影加深，显示五行标签 |
| **选中（第 1 种）** | 边框变为对应五行色 + 发光效果 + ✓ 角标 |
| **选中（第 2 种）** | 同上，"开始烹饪"按钮震动提示 |
| **已选满（2 种）** | 其余卡片半透明，不可点击（除非取消选择） |
| **取消选中** | 再次点击已选中的食材取消 |

### 7.2 动画清单

| 动画 | 时机 | 效果 |
|------|------|------|
| 食材飞入 | 点击食材 | 食材 emoji 从卡片飞入锅中 |
| 锅冒烟 | 点击"开始烹饪" | 锅中冒泡 + 蒸汽特效（2 秒） |

## 9. 技术方案

### 9.1 技术栈

| 层 | 选择 | 原因 |
|----|------|------|
| HTML | 纯 HTML5 | 无需构建工具 |
| CSS | Tailwind CSS（CDN） | 快速开发，响应式 |
| JS | 原生 ES6 | 零依赖，轻量 |
| 数据 | `data.js` 静态文件 | 食材库、菜谱规则 |
| 路由 | 无（单页） | 简单直给 |
| 存储 | `localStorage` | 收藏/成就持久化 |
| 部署 | 静态（双击打开） | 无需服务器 |

### 9.2 文件结构

```
magic-kitchen/
```

### 9.3 菜名生成引擎（核心算法）

```javascript
// 伪代码
function generateDishName(food1, food2) {
  // 1. 先查经典名菜表（硬编码）
  const classic = CLASSIC_DISHES.find(
    d => d.includes(food1.id) && d.includes(food2.id)
  );
  if (classic) return classic.name;

  // 2. 五行相生创意名
  if (isShengRelation(food1.wuxing, food2.wuxing)) {
    return `${food1.name}${food2.name}${WUXING_SHENG_VERBS[food1.wuxing]}`;
  }

  // 3. 同色系创意名
  if (food1.color === food2.color) {
    return `双${food1.colorName}${food1.name.slice(0,2)}${food2.name.slice(0,2)}`;
  }

  // 4. 通用生成
  return `${food1.name.slice(0,2)}炒${food2.name.slice(0,2)}`;
}
```
